Contentora sends transactional emails for account verification and password resets using Nodemailer with React Email templates.

## SMTP Setup

Configure SMTP in the admin panel:

1. Go to **Admin > Settings**
2. Fill in the SMTP fields:

| Setting | Description | Example |
| --- | --- | --- |
| SMTP Host | Mail server hostname | `smtp.gmail.com` |
| SMTP Port | Mail server port | `587` |
| SMTP User | Authentication username / sender email | `noreply@example.com` |
| SMTP Password | Authentication password or app password | — |

## Provider Examples

### Gmail

```
Host: smtp.gmail.com
Port: 587
User: your-email@gmail.com
Password: (App Password — not your Gmail password)
```

Generate an app password at [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords). Requires 2FA enabled.

### Amazon SES

```
Host: email-smtp.us-east-1.amazonaws.com
Port: 587
User: (SMTP username from SES console)
Password: (SMTP password from SES console)
```

### Resend

```
Host: smtp.resend.com
Port: 587
User: resend
Password: re_YOUR_API_KEY
```

### Mailgun

```
Host: smtp.mailgun.org
Port: 587
User: postmaster@your-domain.com
Password: (your Mailgun SMTP password)
```

## Email Templates

Contentora sends two types of emails:

### Verification Email

Sent when a user registers. Contains a verification link that points to:

```
{CLIENT_URL}/verify?token={token}
```

### Password Reset Email

Sent when a user requests a password reset. Contains a reset link:

```
{CLIENT_URL}/reset-password/{token}
```

Both templates use React Email components with a shared layout that includes the site name and branding.

## Testing

Without SMTP configured, email sending will fail silently in development. Check the engine logs for errors.

For local development, you can use [Mailtrap](https://mailtrap.io) or Mailhog to capture emails without sending them to real addresses.
